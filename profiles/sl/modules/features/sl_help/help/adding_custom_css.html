Adding your own custom css

Occasionally you will be called ad to custom css to your site. For vendors. advertisers etc. Here is simple reference guide on adding the that css easily and uploading it to Pantheon with out using git. You will need to know how to FTP and of course css.

CSS definitions for custom styles can be added (though be careful!) in the stylesheet named "reusable.css" in the theme CSS directory (e.g. /sites/all/themes/omega_emgrif/css/reusable.css). At the bottom of the file you will find the custom css area (as seen below), please add your custom css here. It is best practice to test the css on your testing or dev environment before pushing the new css to you live site.

Pantheon & Git

Pantheon uses git for managing code and deployments.

Here are the steps to getting a local codebase copy for your work:

  1. If you don't yet have git installed on your computer, you can download it here for Windows and Mac OS X. If you're running Linux, you should probably install from your distribution's package manager.
  2. Pantheon will need you to upload your SSH public key. If you don't know what an SSH public key is, please follow these key generation instructions to make one to upload. You only need to do this once, no matter how many sites you have on Pantheon.
  3. Give the system a few minutes to reconfigure with your ssh key
  4. Once you've done the first two steps, copy and paste the "git clone" command from the text field at the top of the code log to get a local working copy.

Once you have a local git clone, here's the basic flow for pushing changes to the development environment (be sure to start from within the directory git created):

  1. git add .
  2. git commit -am "Description of the change."
  3. git push origin master
  4. Check the development site for the changes within 5-10 seconds.

Further documentation for getting started with git as well as using the Pantheon workflow are available in the helpdesk. The website "How to Build a Module" offers serveral drupal specific videos on installing git and working with command line.
